Analysing accesibility for Quality Assurance Michigan events Website

Run through each page with the WAVE Chrome Extension.

  • The page has 30 errors wave errors.
  • 29 links are inaccurately linke.
  • There is one broken skip link.

    • Solution: If a link contains no text, the function or purpose of the link will not be presented to the user. This can introduce confusion for keyboard and screen reader users. They should remove the empty link or provide text within the link that describes the functionality and/or target of that link.


    Users should be able to navigate through content using their keyboard

  • The page is accessible through tab indexing and can navigate to pages

    • Users should be able to navigate content using a screen reader.

  • The voiceover does not work effectively on this website. Most of the voiceover only gives code indication of the web content you are hovering over.

    • Solution: They can make the website more accessible and easy to use for screen readers by adding alt text to images and also adding aria labels to the content


    The general architecture and hierarchy of the content should make sense.

  • There is a consistency in maintenance of how headers are placed and how tittle and subtitles are laid out.

    • Charts and images should all have alt-text so that users with screen readers or users on a slow connection will still be able to understand the images.

  • The page does not have a good alt text or navigation. The voice over is broken and as suggested earlier the page needs more aria labels and also alt text for images.
